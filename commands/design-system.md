# デザインシステム確立プロセス

あなたは、指定されたディレクトリ `{{directory}}` のアプリケーションに最適なデザインシステムを確立するエキスパートです。

## 前提条件の確認

まず、以下を確認してください:
1. `.claude/docs/4_frontend_design.md` の内容を確認し、既存のフロントエンドデザイン指針を理解する
2. `{{directory}}/` ディレクトリの構造を確認する
3. 既存のデザイン関連ファイル（CSS、Tailwind設定など）を確認する

## インタラクティブな質問プロセス

以下の順序で、ユーザーに質問しながらデザインシステムを確立していきます。**AskUserQuestion ツールを積極的に使用してください。**

### フェーズ1: デザインコンセプトの確立

1. **プロジェクトの目的とターゲットユーザー**
   - このアプリケーションは何の問題を解決するか？
   - 主なターゲットユーザーは誰か？（年齢層、職業、技術レベルなど）

2. **美的方向性（トーン）の選択**
   - 以下から選択、または組み合わせ:
     - 徹底的にミニマル
     - マキシマリスト
     - レトロフューチャー
     - オーガニック
     - ラグジュアリー
     - 遊び心
     - エディトリアル
     - ブルータリスト
     - アールデコ
     - ソフト/パステル
     - インダストリアル

3. **差別化要素**
   - このアプリを忘れられないものにする要素は何か？
   - ユーザーが覚えている「一つのこと」は何であるべきか？

### フェーズ2: 視覚的要素の定義

#### 2.1 タイポグラフィシステム

以下を質問:
- **フォントの選択**
  - ディスプレイフォント（見出し用）の候補
  - ボディフォント（本文用）の候補
  - Google Fonts、Adobe Fonts、カスタムフォントのどれを使うか

- **タイプスケール**
  - モジュラースケールの比率（1.2倍、1.25倍、1.333倍、1.5倍など）
  - 基準フォントサイズ（通常16px）

- **文字サイズの階層**
  - 見出しレベル（H1〜H6）のサイズ
  - 本文、小さめのテキスト、キャプションのサイズ

#### 2.2 カラーシステム

以下を質問:
- **プライマリカラー**
  - ブランドを代表するメインカラー
  - 鋭いアクセントを持つ支配的な色を選択

- **セカンダリカラー**
  - プライマリをサポートする色

- **ニュートラルカラー**
  - 背景、テキスト、ボーダーなどに使用する無彩色の階層

- **セマンティックカラー**
  - Success（成功）
  - Warning（警告）
  - Error（エラー）
  - Info（情報）

- **ダークモード対応**
  - ダークモードを実装するか？
  - ライトモードと同じ美的方向性を維持するか？

#### 2.3 スペーシングシステム

以下を質問:
- **グリッドシステムの基準**
  - 4px、8px、または別の基準単位を使用するか

- **スペーシングスケール**
  - 推奨: 4px単位（4, 8, 12, 16, 24, 32, 48, 64, 96, 128）
  - または8px単位（8, 16, 24, 32, 40, 48, 64, 80, 96, 128）

#### 2.4 レイアウトグリッド

以下を質問:
- **グリッドカラム数**
  - 12カラム、16カラム、または独自のグリッド

- **ブレークポイント**
  - モバイル（〜640px）
  - タブレット（641px〜1024px）
  - デスクトップ（1025px〜）
  - ワイド（1280px〜、1536px〜など）

- **空間構成のアプローチ**
  - 予測可能な左右対称レイアウト
  - 非対称・重なり・斜めの流れなど実験的レイアウト

#### 2.5 シャドウ・デプスシステム

以下を質問:
- **エレベーションレベル**
  - 平面（影なし）
  - わずかに浮く（subtle）
  - 中程度の浮き（medium）
  - 強い浮き（high）
  - 最も強い浮き（highest）

#### 2.6 ボーダー・角丸

以下を質問:
- **角丸の基準**
  - なし（0px）
  - 小（2-4px）
  - 中（8-12px）
  - 大（16-24px）
  - 完全な丸（9999px or 50%）

- **ボーダーの太さ**
  - 標準のボーダー幅（1px、2px、4pxなど）

#### 2.7 アイコノグラフィ

以下を質問:
- **アイコンのスタイル**
  - アウトライン（線画）
  - 塗りつぶし（filled）
  - 両方を使い分ける

- **アイコンライブラリ**
  - Heroicons
  - Font Awesome
  - Material Icons
  - Lucide
  - その他

- **アイコンサイズ**
  - 小（16px）、中（24px）、大（32px）など

#### 2.8 モーション・アニメーション

以下を質問:
- **アニメーション使用方針**
  - 控えめ（必要最小限）
  - 適度（主要なインタラクションに使用）
  - 大胆（ブランド体験の中核として使用）

- **トランジション時間**
  - 高速（150ms）
  - 通常（300ms）
  - ゆっくり（500ms〜）

- **イージング関数**
  - Linear
  - Ease-in-out
  - Ease-out
  - Custom cubic-bezier

#### 2.9 背景・視覚的詳細

以下を質問:
- **背景スタイル**
  - 単色
  - グラデーション
  - テクスチャ・パターン
  - 画像

- **装飾要素**
  - 幾何学的図形
  - イラストレーション
  - アイコン的装飾
  - なし（ミニマル）

### フェーズ3: コンポーネント定義

以下の主要コンポーネントのスタイル指針を確認:

- **ボタン**
  - プライマリ、セカンダリ、テキストボタンのスタイル
  - ホバー、フォーカス、アクティブ、無効状態

- **フォーム要素**
  - インプット、テキストエリア、セレクト
  - バリデーション表示（エラー、成功）

- **カード**
  - パディング、シャドウ、ボーダー

- **ナビゲーション**
  - ヘッダー、サイドバー、フッタースタイル

- **モーダル・ダイアログ**
  - 背景オーバーレイ、コンテナスタイル

### フェーズ4: アクセシビリティ・パフォーマンス確認

以下を確認:
- **色のコントラスト比**: WCAG 2.1 AA準拠（4.5:1以上）
- **タッチターゲットサイズ**: 最低44×44px（モバイル）
- **フォーカス表示**: キーボードナビゲーション対応
- **パフォーマンス**: Core Web Vitals対応の方針

## 最終アウトプット

すべての質問への回答を基に、以下のファイルを生成してください:

### 1. デザインシステムドキュメント
**保存先**: `{{directory}}/docs/design-system.md`

以下のセクションを含む:
- プロジェクト概要
- デザインコンセプト（目的、トーン、差別化）
- タイポグラフィシステム
- カラーパレット
- スペーシングシステム
- レイアウトグリッド
- シャドウ・デプス
- ボーダー・角丸
- アイコノグラフィ
- モーション・アニメーション
- 背景・視覚的詳細
- コンポーネントガイドライン
- アクセシビリティ基準
- 使用例とベストプラクティス

### 2. Tailwind CSS設定ファイル
**保存先**: `{{directory}}/tailwind.config.js`（既存の場合は更新提案）

確立したデザインシステムを反映:
- カラーパレット
- フォントファミリー
- スペーシングスケール
- ブレークポイント
- シャドウ
- ボーダー半径
- アニメーション設定

### 3. CSSカスタムプロパティファイル（オプション）
**保存先**: `{{directory}}/resources/css/design-tokens.css`

CSS変数として定義:
```css
:root {
  /* カラー */
  --color-primary: ...;
  --color-secondary: ...;

  /* タイポグラフィ */
  --font-display: ...;
  --font-body: ...;

  /* スペーシング */
  --spacing-xs: ...;
  --spacing-sm: ...;

  /* その他 */
}

[data-theme="dark"] {
  /* ダークモード用の上書き */
}
```

### 4. Vueコンポーネントサンプル（オプション）
**保存先**: `{{directory}}/resources/js/components/DesignSystemShowcase.vue`

確立したデザインシステムのショーケースコンポーネント:
- すべての色、タイポグラフィ、スペーシングの表示
- 主要コンポーネントの使用例
- インタラクティブなプレビュー

## 重要な注意事項

1. **4_frontend_design.mdの指針を厳守**
   - 汎用的なフォント（Inter, Roboto, Arial）は避ける
   - 陳腐な配色（紫のグラデーション）は避ける
   - 特徴的で忘れられないデザインを目指す

2. **一貫性**
   - すべての要素が選択した美的方向性と一致する
   - デザインシステム全体が調和している

3. **実用性**
   - 開発者が実際に使いやすいシステム
   - Tailwind CSSやVue 3との統合が容易

4. **拡張性**
   - 将来的な追加・変更に対応できる柔軟性

まず、`{{directory}}/` ディレクトリの構造と既存のデザイン関連ファイルを確認させてください。その後、フェーズ1の質問から始めます。
