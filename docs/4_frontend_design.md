# フロントエンドデザイン指針

## 概要
このフロントエンドデザイン指針は、「特徴的で、本番環境に対応したフロントエンドインターフェースを作成し、汎用的な『AI生成風』の美学を避ける」ことを目的としています。

## デザイン思考プロセス

コーディングの前に、以下を考慮して大胆な美的方向性を理解しコミットする:

- **目的**: これは何の問題を解決し、誰が使用するのか?
- **トーン**: 極端な美的方向性を選択する（徹底的にミニマル、マキシマリスト、レトロフューチャー、オーガニック、ラグジュアリー、遊び心、エディトリアル、ブルータリスト、アールデコ、ソフト/パステル、またはインダストリアル）
- **制約**: フレームワーク、パフォーマンス、アクセシビリティなどの技術要件
- **差別化**: 「何がこれを忘れられないものにするのか? 誰かが覚えている一つのことは何か?」

この指針は強調します: 「明確な概念的方向性を選択し、それを精密に実行する。」

## 美的ガイドライン

5つの主要な領域に焦点を当てる:

1. **タイポグラフィ**: 汎用的なオプションではなく、美しく特徴的なフォントを選択する。特徴的なディスプレイフォントと洗練されたボディフォントをペアにする。

2. **カラー & テーマ**: CSS変数を使用して一貫性のある美学を開発する。「鋭いアクセントを持つ支配的な色は、臆病で均等に分散されたパレットを上回る。」

3. **モーション**: 高インパクトな瞬間にアニメーションを使用する。HTMLにはCSSのみ; Reactには利用可能な場合Motionライブラリを使用。

4. **空間構成**: 予期しないレイアウト、非対称性、重なり、斜めの流れ、およびグリッドを壊す要素を採用する。

5. **背景 & 視覚的詳細**: テクスチャ、グラデーション、パターン、および美的に一致する装飾要素を通じて雰囲気を作る。

## 重要な回避リスト

以下は決して使用しない: 汎用フォント（Inter, Roboto, Arial）、陳腐な配色（紫のグラデーション）、予測可能なレイアウト、またはコンテキスト固有の特徴に欠けるクッキーカッターデザイン。

## 実装の原則

### 1. デザインシステムの一貫性
- CSS変数を使用してカラーパレット、スペーシング、タイポグラフィを定義
- 再利用可能なコンポーネントを作成し、デザインの一貫性を保つ
- Tailwind CSSのユーティリティクラスを活用しつつ、独自性を失わない

### 2. レスポンシブデザイン
- モバイルファーストアプローチを採用
- ブレークポイントで美的方向性を維持
- デバイスごとに適切なタッチターゲットサイズを確保

### 3. アクセシビリティ
- WCAG 2.1 AA準拠を目指す
- 適切な色のコントラスト比を確保
- キーボードナビゲーションをサポート
- スクリーンリーダー対応

### 4. パフォーマンス
- 画像の最適化（WebP、遅延読み込み）
- CSSアニメーションはGPUアクセラレーションを活用
- 不要なJavaScriptの読み込みを削減
- Core Web Vitalsの最適化

### 5. ダークモード対応
- プロジェクト全体でダークモードをサポート
- `dark:`プレフィックスを使用して適切なスタイルを適用
- カラーパレットはライト/ダークモード両方で視認性を確保

## デザインレビューチェックリスト

実装前に以下を確認:
- [ ] 明確な美的方向性が選択されているか
- [ ] 既存のデザインシステムとの一貫性があるか
- [ ] 汎用的なフォントや配色を避けているか
- [ ] アクセシビリティ要件を満たしているか
- [ ] レスポンシブデザインが適切に実装されているか
- [ ] パフォーマンスへの影響を考慮しているか
- [ ] ダークモード対応がなされているか
